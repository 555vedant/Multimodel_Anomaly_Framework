================================================================================
DRIFT-AWARE RELATIONAL ANOMALY DETECTION IN MULTIVARIATE SENSOR SYSTEMS
PROJECT REPORT & CONTEXT DOCUMENT
================================================================================

1. PROJECT OVERVIEW & ARCHITECTURE
We built an unsupervised anomaly detection system for the UCI Gas Sensor Array 
Drift Dataset (16 sensors x 8 features = 128 features over 36 months).
The goal: Detect natural hardware drift as sensors degrade over time.
We use Batches 1-5 for training (learning "normal") and Batches 6-10 for testing.

WHAT IS AN ANOMALY IN THIS CONTEXT?
An anomaly here refers to data points where the sensor output diverges from the established
"normal" profile learned during the initial 36 months of sensor life (Batches 1-5). 
These aren't necessarily spikes caused by a different gas, but rather represent 
"sensor drift" - physical degradation of the sensors resulting in shifted or broken
response patterns compared to their fresh state.

The system uses a MULTI-MODEL detection approach combined as S = α*T + β*R.

2. DETAILED MODEL OPERATION

A) TEMPORAL PATH (Autoencoder)
   - Purpose: Detects when individual sensor readings fall outside normal ranges.
   - Architecture: 128 -> 64 -> 32 (Latent Bottleneck) -> 64 -> 128. 
     (Uses BatchNorm and Dropout=0.2 for regularization).
   - How it works: The Autoencoder is trained only on Batches 1-5. It learns to 
     compress the 128 features into a 32-dimensional space and reconstruct them.
     It becomes very good at reconstructing "normal" data. When an abnormal sample 
     arrives (e.g. degraded sensor), the bottleneck prevents accurate reconstruction.
   - Error Metric: Mean Squared Error (MSE) between the input vector and the 
     reconstructed vector. High MSE means high temporal anomaly score.

B) RELATIONAL PATH (Correlation Pairs)
   - Purpose: Detects when the historical relationship between two sensors breaks, 
     even if their individual values seem normal to the Autoencoder.
   - How it works: 
     1. During training, computes the Pearson correlation matrix for all 128x128 pairs.
     2. Selects strongly correlated pairs (|r| >= 0.7), capped at 500 pairs.
     3. For each pair (i, j), fits a simple Ordinary Least Squares (OLS) linear regression: 
        Feature_j = (Slope * Feature_i) + Intercept. It also records the training 
        residual standard deviation.
     4. During testing, it uses feature i to predict feature j. 
   - Error Metric: It calculates the z-scored residual: 
     |Actual_j - Predicted_j| / Train_Residual_Std. 
     The final relational error for a sample is the mean of squared z-scores across 
     all tracked pairs.

3. CORE CONFIGURATION & SCORING LOGIC
Scores are normalized to [0,1] then combined: S = (alpha * Temporal_Norm) + (beta * Relational_Norm)
- ALPHA: Weight for autoencoder MSE (default 0.6).
- BETA: Weight for relational error (default 0.4).
- CORR_THRESHOLD: Minimum correlation to track a pair (default 0.7).
- THRESHOLD_METHOD: Determines the anomaly cutoff using training scores 
  (e.g., mean + 2*std or 98th percentile).

4. EXPERIMENT RESULTS SUMMARY
Recorded in outputs/experiment_comparison.csv:

  [Exp 1: Baseline] (exp_baseline.json)
  - Settings: alpha=0.6, beta=0.4, threshold=mean_std, corr_threshold=0.7
  - Anomalies Detected: 234 / 10277 (2.28%)
  - Drift Trend: Anomalies exclusively peaked in Batch 10 (232 anomalies, 6.4% rate), 
    confirming severe late-stage sensor drift.

  [Exp 2: Sensitive] (exp_sensitive.json)
  - Settings: alpha=0.55, beta=0.45, threshold=percentile (98th), corr_threshold=0.65
  - Anomalies Detected: 237 / 10277 (2.31%)
  - Outcome: Slightly more weight on relationships; detected marginally more anomalies.

  [Exp 3: Conservative] (exp_conservative.json)
  - Settings: alpha=0.7, beta=0.3, threshold=mean_std, corr_threshold=0.75
  - Anomalies Detected: 288 / 10277 (2.80%)
  - Outcome: Unexpectedly higher anomalies flagged due to strict pair limits.

5. CONCLUSION
The combined Temporal + Relational approach maps natural sensor degradation effectively.
Early test batches (6, 7) show near 0% anomaly rates, while the final batch (10) 
exhibits widespread failures, validating that long-term sensor drift is captured.
